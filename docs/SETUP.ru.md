# Инструкция по запуску и настройке

Это руководство охватывает всё — от установки до тонкой настройки оптической геометрии. Для общей картины сначала прочитайте [главный README](../README.ru.md).

---

## 1. Требования

**Компьютер (запускает сервер):**
- Python **3.8+** (только стандартная библиотека — для сервера `pip install` не нужен)
- **OpenSSL** в `PATH` (используется один раз для создания самоподписанного сертификата)

**Телефон (выполняет оптику):**
- Экран **OLED / AMOLED** (LCD работает плохо — именно попиксельное свечение OLED заставляет канал работать)
- **Фронтальная камера**
- Современный мобильный браузер (Chrome / Safari)

**Сеть:** телефон и компьютер должны быть в **одной локальной сети Wi‑Fi**.

**Оптика:** небольшое плоское зеркало (≈ 10 × 10 см). Лучше всего первоповерхностное; для первого запуска подойдёт и зеркало из ванной.

---

## 2. Установка

```bash
git clone https://github.com/infosave2007/svetoch.git
cd svetoch/app
python3 server.py
```

При первом запуске сервер:

1. Генерирует самоподписанный TLS‑сертификат (`cert.pem` / `key.pem`) — HTTPS **обязателен**, потому что браузеры дают доступ к камере только на защищённых источниках.
2. Определяет ваш IP в локальной сети и слушает `0.0.0.0:8443`.
3. Выводит адреса телефона и админки.

```
==================================================
  NVG Optical Computer — Server
==================================================

📱 Телефон: https://192.168.x.x:8443/
🖥️  Админка: https://192.168.x.x:8443/admin

Всё на одном порту 8443 — без WebSocket!
==================================================
```

> Сгенерированные `cert.pem` / `key.pem` — локальные секреты и **исключены из git**. Удалите их, чтобы пересоздать.

## 3. Физическая установка

```
    ┌────────────────────────────────────┐
    │  [фронт. камера]     AMOLED экран   │  ← телефон, экраном ВНИЗ
    └────────────────────────────────────┘
                    │
                    │  d ≈ 3–5 см   (стопка книг / подставка)
                    │
    ╔════════════════════════════════════╗
    ║         ЗЕРКАЛО  10×10 см           ║  ← ровно на столе
    ╚════════════════════════════════════╝
```

| Компонент | Примечание |
|-----------|------------|
| Телефон | OLED‑экран вниз, фронтальная камера смотрит на зеркало |
| Зеркало | Плоское, чистое, прямо под камерой |
| Зазор `d` | Начните с 3–5 см. Расстояние самоизображения Тальбота зависит от `d`; ~37 мм — удачная точка на эталонном устройстве |
| Свет | Работайте в **тёмной, стабильной** комнате. Внешний свет — это шум |

Советы:
- Держите зазор **жёстким** — даже малые вибрации размывают интерференционный паттерн.
- Минимальная дистанция фокусировки камеры важна (≈ 10 см на Xiaomi 12 Lite); зеркало удваивает оптический путь, что отчасти и объясняет его необходимость.
- Пыль и отпечатки на зеркале или объективе напрямую снижают контраст.

## 4. Запуск экспериментов

1. **Телефон:** откройте адрес *Телефон*. Примите предупреждение о сертификате («Дополнительно → Перейти»). Дайте доступ к **камере**. Не давайте экрану гаснуть (отключите автоблокировку).
2. **Админка:** откройте адрес *Админка* на компьютере. Вы увидите все четыре семейства экспериментов, автоматически загруженные из `app/stages/`.
3. Выберите отдельные эксперименты или **все**, задайте язык интерфейса (RU / EN) и нажмите **Старт**.
4. Телефон опрашивает сервер, выполняет выбранные эксперименты и передаёт логи, живые метрики и графики в панель.
5. По завершении прогон сохраняется в `logs/nvg_poc_v3_<метка_времени>.json`. Любой прошлый прогон можно скачать из панели истории.

Режим **Студент** (`/student`) отражает текущую сессию в упрощённом виде «только просмотр» — для демонстраций и занятий.

## 5. Настройка

Файла конфигурации нет — система настраивается тем, **что вы положили в `app/stages/` и какие эксперименты выбрали**. Немного серверных параметров:

| Что | Где | По умолчанию |
|-----|-----|--------------|
| Порт | `server.py` → `HTTPServer(("0.0.0.0", 8443), …)` | `8443` |
| Язык интерфейса по умолчанию | `server.py` → `state["default_lang"]` (или `/api/set_lang?lang=en`) | `ru` |
| Куда сохраняются прогоны | `server.py` пишет в `../logs` относительно `app/` | `svetoch/logs/` |
| Subject сертификата | `generate_cert()` `-subj` | `/CN=NVG-PoC` |

**Категории экспериментов** — это просто папки в `app/stages/`. Их отображаемые названия берутся из `info.json` в каждой папке:

```json
{ "ru": "Нейронные сети и Трансформеры",
  "en": "Neural Networks & Transformers" }
```

## 6. Как добавить свой эксперимент

Каждый эксперимент — это два файла в папке категории:

```
app/stages/1_neural_networks/
├── stage99_myexp.js      ← логика эксперимента (ES‑модуль)
└── stage99_myexp.json    ← двуязычные метаданные
```

`.json` управляет каталогом и интерфейсом:

```json
{
  "ru": {
    "name": "Моё название",
    "description": "Краткое описание.",
    "description_all": "Полное объяснение: физический принцип, алгоритм, метрика."
  },
  "en": {
    "name": "My title",
    "description": "Short description.",
    "description_all": "Full explanation: physical principle, algorithm, metric."
  },
  "mirrorless": false
}
```

- Числовой префикс (`stage99`) задаёт порядок внутри категории.
- `mirrorless: true` помечает контрольный эксперимент, который работает и без зеркала.
- После добавления файлов пересоберите каталог: `python ../scripts/gen_experiments.py`.

Сервер обнаружит новый эксперимент автоматически — менять логику не нужно, просто перезагрузите панель.

## 7. Программная валидация (телефон не нужен)

Папка [`simulation/`](../simulation/) воспроизводит ключевые результаты на NumPy — полезно для понимания математики и для CI.

```bash
pip install numpy
python simulation/xiaomi_12lite_llm_calc.py   # полный бюджет параметров
python simulation/stage1_optical_channel.py   # модель оптического канала
python simulation/stage2_dot_product.py       # оптическое скалярное произведение
python simulation/stage3_single_layer.py      # один слой нейросети, оптика vs цифра
python simulation/stage4_full_inference.py    # авторегрессивный оптический LLM
```

## 8. Устранение неполадок

| Симптом | Вероятная причина и решение |
|---------|------------------------------|
| Камера не запрашивает разрешение | Вы открыли `http://`, а не `https://`, или отклонили сертификат. Откройте HTTPS‑адрес заново и примите предупреждение. |
| «Телефон отключён» в админке | Телефон и компьютер в разных сетях, или брандмауэр блокирует порт 8443. Подключите оба к одному Wi‑Fi; откройте порт. |
| Все метрики не проходят / нет контраста | Слишком много внешнего света, зеркало смещено или экран не направлен на зеркало. Затемните комнату, отцентрируйте зеркало под камерой. |
| Низкий динамический диапазон (~1.0) | Это в точности сигнатура **контроля без зеркала** — проверьте, что зеркало действительно отражает экран в объектив. |
| Экран гаснет посреди прогона | Отключите автоблокировку; держите вкладку браузера на переднем плане. |
| Ошибки сертификата при каждой загрузке | Самоподписанные сертификаты — это ожидаемо; примите один раз. Некоторые браузеры требуют ввести `thisisunsafe` на странице предупреждения. |
| Размыто / нет фокуса | Слегка увеличьте зазор; очистите объектив; убедитесь, что соблюдена минимальная дистанция фокусировки (путь через зеркало помогает). |

---

## 9. О безопасности

Сервер использует **самоподписанный** сертификат и привязывается к вашей локальной сети. Он предназначен для доверенной локальной сети во время экспериментов, а не для выхода в публичный интернет. Не коммитьте `cert.pem` / `key.pem` (они уже исключены из git).
