## Что это?

**Светоч** — это proof‑of‑concept, который выполняет настоящие вычисления **в свете, а не в кремнии**.

OLED‑экран телефона отображает операнды как паттерны яркости. Свет отражается от обычного карманного зеркала и улавливается фронтальной камерой. Поскольку камера *интегрирует* падающие фотоны за время экспозиции, слияние света на сенсоре физически выполняет сложения и скалярные произведения — базовую операцию любой нейросети — за один оптический такт.

Поверх этого единственного оптического примитива построена башня из **101 эксперимента**: от калиброванного оптического канала и скалярных произведений до полноценных слоёв трансформера и авторегрессивной генерации текста, а затем дальше — к аналогам из волновой физики, квантовых вентилей и классических алгоритмов.

> **Стоимость:** Минимальная — нужно лишь обычное зеркало. **Оборудование:** любой современный телефон с OLED/AMOLED‑экраном и фронтальной камерой. **Эталонное устройство:** Xiaomi 12 Lite.

```
    ┌────────────────────────────────────┐
    │  [фронт. камера]     AMOLED экран   │  ← телефон, экраном ВНИЗ
    └────────────────────────────────────┘
                    │  ~5 см
    ╔════════════════════════════════════╗
    ║         ЗЕРКАЛО  10×10 см          ║  ← лежит на столе
    ╚════════════════════════════════════╝
```

Экран показывает веса → зеркало возвращает свет → камера читает результат. Этот «полёт света туда и обратно» *и есть* вычисление.

## Почему это важно

- **Это реальность, а не симуляция.** Приложение управляет настоящим оптическим каналом «экран → зеркало → камера» и измеряет то, что видит сенсор.
- **Встроенный контрольный эксперимент.** Каждое утверждение проверяется прогоном *без зеркала*: когда экран направлен в открытую комнату, вычислительный канал исчезает (динамический диапазон 1.42 → 1.00, контраст слабее в ×4000) — это доказывает, что эффект создаётся оптикой, а не программными артефактами.
- **Воспроизводимость.** Все 101 эксперимент скриптуемы, у метрик есть явные пороги прохождения, а каждый прогон сохраняется в JSON.
- **Открытость для доработки.** Каждый эксперимент — это маленький самодостаточный JavaScript‑модуль в `app/stages/`. Добавить эксперимент — значит просто положить туда `.js` и двуязычный `.json`.

## Как это работает (архитектура)

В системе три роли, общающиеся через один порт HTTPS:

| Роль | URL | Что делает |
|------|-----|------------|
| **Телефон** | `https://<ip>:8443/` | Рисует паттерны на OLED, снимает кадры камерой, выполняет эксперименты, отправляет результаты. |
| **Админка** | `https://<ip>:8443/admin` | Выбирает эксперименты, нажимает Старт/Стоп, смотрит живые логи и графики, скачивает JSON результатов. |
| **Студент** | `https://<ip>:8443/student` | Режим «только просмотр» для класса: живое зеркало экрана телефона, текущий этап с прогресс‑баром, карта всех экспериментов, поток логов, таблица результатов (пройдено/не пройдено) и пояснения по тапу на этап. Не запускает и не останавливает прогон — это остаётся за админкой. |

```
   Админ-панель  ──HTTP──►  server.py  ◄──HTTP опрос──  Телефон
   (выбор и старт)          (общее состояние)          (выполняет оптику)
                                  │
                                  └── сохраняет каждый прогон → logs/*.json
```

`server.py` — это HTTP‑сервер на чистом Python без зависимостей. Он хранит общее состояние, передаёт команды Старт/Стоп телефону (который их опрашивает), собирает результаты по этапам и записывает весь прогон в `logs/`. Никаких WebSocket, фреймворков и шагов сборки.

Эксперименты **обнаруживаются автоматически**: сервер сканирует `app/stages/<категория>/stage*.js`, читает соответствующий `stage*.json` с двуязычными названиями/описаниями и отдаёт их через `/api/stages`. Положите новую пару файлов в папку категории — и она появится в панели.

## Быстрый старт

> Полные детали, устранение неполадок и настройка — в **[инструкции по запуску](docs/SETUP.ru.md)**.

**Требования:** Python 3.8+, OpenSSL (для самоподписанного сертификата), телефон и компьютер в **одной сети Wi‑Fi** и небольшое зеркало.

```bash
git clone https://github.com/infosave2007/svetoch.git
cd svetoch/app
python3 server.py
```

При первом запуске сервер сгенерирует самоподписанный сертификат и выведет два адреса:

```
📱 Телефон: https://192.168.x.x:8443/
🖥️  Админка: https://192.168.x.x:8443/admin
🎓 Студент: https://192.168.x.x:8443/student
```

1. **На телефоне:** откройте адрес *Телефон*, примите предупреждение о самоподписанном сертификате, дайте доступ к **камере** и не давайте экрану гаснуть. Положите телефон экраном вниз на высоте ~5 см над зеркалом.
2. **На компьютере:** откройте адрес *Админка*, выберите эксперименты (или «все») и нажмите **Старт**.
3. Наблюдайте за метриками в реальном времени. Каждый завершённый прогон сохраняется в `logs/nvg_poc_v3_<метка_времени>.json` и его можно скачать из панели.
4. **Для класса / аудитории (опционально):** откройте адрес *Студент* на проекторе или втором экране — он зеркалит прогон в режиме «только просмотр» (экран, прогресс, результаты, пояснения) без каких‑либо органов управления.

### ⚠️ Два обязательных условия на телефоне

**1. Обязательно включите полноэкранный режим.** Эксперименты рисуют паттерны на весь экран OLED. Если видны адресная строка и панели браузера — геометрия и контраст ломаются, и метрики не проходят. На странице *Телефон* **коснитесь экрана, чтобы войти в полноэкранный режим** (если прогон запущен из админки, специально появляется оверлей «коснитесь, чтобы начать» — полноэкранный режим браузер разрешает включить только по касанию). И отключите автоблокировку, чтобы экран не гас посреди прогона.

**2. Добавьте свой телефон (чтобы оптика была откалибрована под него).** Геометрия зависит от шага пикселя OLED вашего телефона. Выберите устройство из выпадающего списка на странице *Телефон*; если его там нет — добавьте в [`app/devices.json`](app/devices.json):

```json
"my_phone": {
  "name": "Мой телефон",
  "oled_pitch_um": 60.0,        // шаг пикселя OLED в мкм  ≈ 25400 / PPI экрана
  "sensor_pitch_um": 0.8,       // размер пикселя фронт-камеры в мкм (из спецификации)
  "recommended_dist_mm": 35.0   // стартовый зазор; растёт примерно как pitch² (Тальбот) — подберите по макс. контрасту
}
```

Перезагрузите страницу *Телефон* — устройство появится в списке (выбор запоминается). Значение `recommended_dist_mm` — это зазор между телефоном и зеркалом: начните с него и подстройте до самых чётких полос. Если вашего телефона нет в списке — это не ошибка, просто оптика не будет откалибрована; эталонные профили (Xiaomi, iPhone, Samsung, Pixel, …) уже включены.

**Нет зеркала или телефона под рукой?** Запустите программную валидацию — см. [`simulation/`](simulation/):

```bash
pip install numpy
python simulation/xiaomi_12lite_llm_calc.py   # параметры системы
python simulation/stage1_optical_channel.py   # оптический канал
python simulation/stage4_full_inference.py    # полный оптический LLM-инференс
```

## Эксперименты вкратце

101 эксперимент в четырёх семействах — полный каталог с физикой и критериями прохождения в **[docs/EXPERIMENTS.ru.md](docs/EXPERIMENTS.ru.md)**.

| Семейство | Кол-во | Примеры |
|-----------|------:|---------|
| 🧠 **Нейросети и Трансформеры** | 22 | Оптический канал, скалярное произведение, MatVec, полный слой LLM, SiLU, softmax, RoPE, MLP, оптическая CNN, LSTM через OLED‑персистентность |
| 🌊 **Волновая физика и Основания** | 29 | Самоизображение Тальбота, интерференция, дифракция, Фабри‑Перо, тепловая конвекция, длина когерентности |
| ⚛️ **Квантовые вентили и Вычисления** | 24 | ψ‑суперпозиция, θ‑тест Белла, Адамар, QFT, Гровер, телепортация, Дойч‑Йожа, CNOT, QEC |
| 🔢 **Математические алгоритмы** | 26 | Пропускная способность канала, томография через transfer‑matrix, fidelity, HOM, QRNG, Шор, прикладной DSP |

Эталонные результаты на Xiaomi 12 Lite (с зеркалом, зазор ~37 мм): оптический канал линеен (R² ≈ 1.0), оптически вычисленный слой нейросети совпадает с цифровым с точностью ~1%, а 2‑слойный трансформер генерирует текст авторегрессивно со скоростью несколько токенов в секунду.

## Как добавить эксперимент

Добавление эксперимента намеренно простое:

1. Создайте `app/stages/<категория>/stageN_token.js` с логикой эксперимента.
2. Создайте `app/stages/<категория>/stageN_token.json` с полями `ru` / `en` (`name`, `description`, `description_all`) и флагом `mirrorless`.
3. Запустите `python scripts/gen_experiments.py`, чтобы обновить каталог.

Сохраняйте стиль нового кода согласованным с существующими модулями той же категории.
